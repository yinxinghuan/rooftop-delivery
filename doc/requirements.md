# Requirements

## 1. Overview

《Rooftop Delivery / 天台速递》是一款手机竖屏 3D 弹道投掷游戏，玩家在黄昏城市屋顶拖动并抛出 8 件包裹，读取风向、控制力度并尽量让包裹落在远处收件标记中心。

## 2. Visual Design

- 游戏画面以 390 × 720 的竖屏构图为设计基准，并等比覆盖不同尺寸的手机屏幕；3D 场景占满全屏，HUD 使用安全区内绝对定位。
- 镜头位于出发屋顶后上方，以约 50° 透视俯看两栋主楼和中间街谷；前景屋顶占画面底部约 28%，目标屋顶位于画面中部，远景由 12–18 栋低多边形建筑形成层次。
- 天空使用黄昏渐变：顶部 `#5B5E8B`、中部 `#F6B38A`、地平线 `#FFD4A3`；建筑主色为 `#554F67`、`#6E6579`、`#8B7580`，屋顶为暖灰米色 `#E9D7B7`。
- 包裹为 0.72 × 0.52 × 0.48 世界单位的珊瑚红纸箱，主色 `#F05D4E`，封箱带 `#F7D58B`，侧面带白色邮政箭头；飞行时每 70 毫秒释放一枚 0.08–0.16 世界单位的纸屑粒子。
- 收件标记由三个同心圆组成：中心直径 1.7 世界单位，颜色 `#F05D4E`；中圈直径 3.4，颜色 `#F7D58B`；外圈直径 5.4，颜色 `#F7F0DC`。圆环以每秒 0.8 弧度缓慢旋转，中心信标上下浮动 0.12 世界单位。
- 标志性视觉元素为动态风带：6–10 条半透明青色 `#79D7D2` 曲线粒子横贯街谷，移动方向与当前风向一致，完整穿越屏幕用时 1.4–2.2 秒；HUD 同时显示风向箭头和 `0.0–1.2` 风力刻度。
- HUD 使用一条深墨紫半透明航线仪表带，背景为 `rgba(39,40,58,0.94)`、模糊 14px、纸张米白边框；包裹同款珊瑚橘 `#F05D4E` 作为界面主色，使 UI 与场景形成统一的航空邮政色板。得分字号不小于 30px，其余标签不小于 11px。显示字体使用 `Arial Black` / `Avenir Next Condensed`，正文使用 `Avenir Next` / 系统无衬线，数据使用 `SFMono-Regular` / 等宽字体。
- HUD 颜色为深墨紫 `#27283A`、纸张米白 `#FFF5DE`、邮政红 `#F05D4E` 和风带青 `#79D7D2`。主要按钮最小高度 52px，左右内边距 28px，阴影为 `0 8px 0 rgba(39,40,58,0.35)`。
- 命中时显示 0.8 秒上飘分数、0.9 秒快递员台词气泡和 18–28 枚纸屑；中心命中附加 0.35 秒圆环冲击波。失误时画面轻微水平抖动 0.28 秒，包裹坠入街谷并缩小消失。
- 开始页直接展示可运动的 3D 城市、目标和风带，标题位于上方；画面下方 18% 处只显示一个向上移动 48px 的手指动画和“向上拖动 · 松手投掷”单行引导。开始页不显示营销副标题、说明段落、票据卡或独立开始按钮，点击开始页任意非排行榜区域即可进入游戏。
- 开始页常驻 48px 高的冠军入口：有数据时显示第一名头像、姓名和分数，无数据时显示“排行榜”；点击后打开最高 440px 的榜单弹层。榜单每行显示名次、30px 圆形头像、姓名和分数，其他玩家行可点击打开 Aigram 主页，自身行显示“你”且不可点击。
- 素材清单：程序生成低多边形建筑、屋顶设施、包裹、收件圆环、风带、纸屑、云层和邮政 UI；平台 Aigram 水印为唯一外部矢量资源。

## 3. Game Mechanics

- 每局包含 8 件包裹，初始失误额度为 3，初始分数与 combo 均为 0。
- 开始后前 1,500 毫秒为缓冲期：显示风向、目标和操作引导，不允许包裹坠落计为失误；缓冲结束后第一件包裹进入可投掷状态。
- 每次投掷前随机生成目标横坐标 `-2.7–2.7`、纵深坐标 `-13.5–-8.5` 世界单位；相邻两次目标中心至少相距 2.2 世界单位。
- 当前风力为 `-1.2–1.2` 世界单位/秒²，正值向右、负值向左；风力在每次投递后重抽，绝对值低于 0.25 的概率为 20%。飞行期间风持续叠加到横向速度。
- 拖动的向上距离映射为 `0–1` 力度：小于 24px 不发射，24–180px 线性映射，超过 180px 截断；横向拖动 `-120–120px` 映射到横向初速度 `-4.2–4.2` 世界单位/秒。
- 包裹竖直初速度为 `5.5 + 力度 × 2.8`，向前初速度为 `6.2 + 力度 × 4.8`，重力为 10.8 世界单位/秒²；角速度根据横向拖动与力度在 `2.4–8.0` 弧度/秒之间生成。
- 包裹首次接触目标屋顶时保留 22% 的竖直速度和 62% 的水平速度，最多弹跳 2 次；首次接触后 1,100 毫秒或总速度低于 0.9 世界单位/秒时结算落点。
- 落点距目标中心不超过 0.85 世界单位为中心命中，基础 100 分；不超过 1.7 为准时送达，基础 60 分；落在目标屋顶但超过 1.7 为屋顶送达，基础 25 分；未落在目标屋顶为失误，扣除 1 次机会且 combo 清零。
- 连续成功从第 2 次开始显示 combo；每次成功额外获得 `min(combo, 5) × 10` 分，中心命中连续 3 次额外奖励 100 分。成功后 combo 加 1，失误后归零。
- 结算后等待 900 毫秒生成下一件包裹、目标和风向；完成第 8 件或失误累计到 3 次时结束本局。
- 主循环以屏幕刷新率运行，物理更新时间步最大截断为 33 毫秒；包裹低于世界高度 `-7` 时立即判定坠落。

## 4. Controls

- Pointer / Touch：在包裹可投掷时按下游戏区域开始瞄准，向上拖动增加力度，左右拖动改变横向初速度，松开立即投掷；拖动不足 24px 时松开取消本次投掷。
- 瞄准期间显示 18 个弹道预测点，预测包含当前风力；力度越大，点的透明度和间距越高。
- Keyboard：按住 `Space` 以每秒 0.75 的速度积累力度，`ArrowLeft` / `ArrowRight` 调整横向瞄准，松开 `Space` 投掷；`R` 在结算页重新开始，`Escape` 返回开始页。
- 所有开始、再来一次和返回首页按钮统一使用 Pointer Down 触发，触控目标高度不小于 44px。

## 5. Win / Lose Conditions

- 成功完成 8 次投递后显示“本轮派送完成”，无论其中是否有屋顶边缘送达。
- 累计 3 次包裹坠楼会提前结束，显示“本轮提前收车”。
- 结算单必须显示最终得分、历史最高分、成功投递数、中心命中数、最高 combo、“再跑一轮”和“返回首页”。
- 历史最高分在每局结束时保存；若本局刷新最高分，结算单显示 `NEW ROUTE RECORD / 新路线纪录` 印章。
- 每局开始前快照玩家在平台榜单中的旧最高分；结算时向永久游戏 UUID `3ecbe73c-4354-438b-8a30-f523370c0324` 提交最终得分。若本局刷新个人平台纪录，只向此次刚被超过且分数最高的一名其他玩家发送 `score_beat` 通知；平分、未破个人纪录和自己均不通知，接口失败不影响结算。
- 非 AlterU 环境不请求排行榜接口；排行榜弹层显示“在 AlterU 中查看排行榜”及指向 `https://alteru.app` 的下载入口。

## 6. Sound Effects

- 开始：两枚正弦波音符 392Hz → 523Hz，各 90ms，间隔 70ms，峰值音量 0.08。
- 开始拖动：三角波 240Hz → 320Hz，持续 70ms，峰值音量 0.035。
- 发射：锯齿波 180Hz → 92Hz，持续 180ms，叠加 520Hz 正弦波 60ms，峰值音量分别为 0.055 和 0.025。
- 屋顶弹跳：三角波 150–210Hz，持续 65ms，每次弹跳音量衰减 45%。
- 普通送达：正弦波 440Hz、554Hz、659Hz 三音依次播放，每音 80ms，峰值音量 0.055。
- 中心命中：正弦波 523Hz、659Hz、784Hz、1047Hz 四音上升，每音 90ms，并叠加 120ms 的低频纸箱撞击声，峰值音量 0.075。
- 坠楼失误：方波 170Hz → 72Hz，持续 300ms，峰值音量 0.045。
- Combo 奖励：从 660Hz 起按 combo 每级增加 70Hz 的 70ms 正弦波提示，最高 940Hz，峰值音量 0.045。
- 游戏结束：完成 8 件时播放 392Hz、523Hz、659Hz、784Hz 的 420ms 上升和弦；失误结束时播放 220Hz → 110Hz 的 480ms 三角波下降音，峰值音量 0.065。
